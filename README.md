## Fonctionnalites

| Categorie | Exemples | Capture |
|-----------|----------|---------|
| Macros de reflexion | `UCLASS`, `USTRUCT`, `UENUM`, `UINTERFACE`, `UPROPERTY`, `UFUNCTION` | `@attribute` |
| Generated body | `GENERATED_BODY()`, `GENERATED_UCLASS_BODY()` | `@attribute` |
| Delegates | `DECLARE_DYNAMIC_MULTICAST_DELEGATE_*`, `DECLARE_EVENT_*` | `@attribute` |
| API Export | `MYGAME_API`, `ENGINE_API`, `CORE_API` | `@attribute` |
| Specifier flags | `EditAnywhere`, `BlueprintReadWrite`, `BlueprintCallable`, `Category`... (60+) | `@constant` |
| Logging | `UE_LOG`, `UE_LOGFMT`, `UE_CLOG` | `@function` |
| Assertions | `check`, `checkf`, `ensure`, `ensureAlways`, `verify` | `@function` |
| Casts UE | `Cast<T>()`, `CastChecked<T>()` | `@function` |
| TEXT macro | `TEXT("...")` | `@function.special` |
| Misc macros | `DEFINE_LOG_CATEGORY`, `IMPLEMENT_MODULE` | `@attribute` |
| Base C/C++ | Coloration standard complete incluse | -- |

## Quick Start

### Prerequis

- **[Zed Editor](https://zed.dev/download)** (derniere version stable)
- **[Rust via rustup](https://rustup.rs/)** (Zed compile les grammaires Tree-sitter en WASM, rustup est obligatoire)

> **Windows** : si Rust est installe via un installeur standalone ou chocolatey, la compilation des grammaires echouera. Utiliser **rustup** exclusivement.

### 1. Cloner le repo

```bash
git clone https://github.com/hoklims/unreal-cpp-zed.git
```

### 2. Installer l'extension dans Zed

1. Ouvrir **Zed**
2. Ouvrir la palette de commandes :
   - **Windows / Linux** : `Ctrl + Shift + P`
   - **macOS** : `Cmd + Shift + P`
3. Taper `zed: install dev extension` et valider
4. Dans le file picker, selectionner le dossier **`unreal-cpp-zed/`** (la racine contenant `extension.toml`)
5. Attendre la compilation de la grammaire (premiere fois uniquement, ~30s)

> Si tout se passe bien, la page **Extensions** affiche `Unreal C++` avec la mention *"Installed (dev)"*.

### 3. Activer sur vos fichiers Unreal

**Option A — Manuelle (test rapide)**

1. Ouvrir un fichier `.h` ou `.cpp` de votre projet UE
2. Cliquer sur **l'indicateur de langage** en bas a droite de la status bar (affiche "C++")
3. Selectionner **"Unreal C++"**

**Option B — Automatique par projet (recommande)**

Creer un fichier **`.zed/settings.json`** a la racine de votre projet Unreal :

```json
{
  "file_types": {
    "Unreal C++": ["cpp", "h", "hpp", "cc", "cxx"]
  }
}
```

Tous les fichiers C++ de ce projet utiliseront automatiquement la coloration Unreal.

**Option C — Global (tous les projets)**

Ouvrir les settings Zed (`Ctrl + ,` / `Cmd + ,`) et ajouter le meme bloc `file_types` dans la config globale.

### 4. Verifier que ca fonctionne

Ouvrir un fichier contenant du code Unreal typique :

```cpp
UCLASS(BlueprintType, Blueprintable)
class MYGAME_API AMyActor : public AActor
{
    GENERATED_BODY()

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Stats")
    float Health = 100.f;

    UFUNCTION(BlueprintCallable, Category = "Combat")
    void TakeDamage(float Amount);
};
```

Vous devriez voir :
- `UCLASS`, `UPROPERTY`, `UFUNCTION`, `GENERATED_BODY` surlignés differemment des fonctions normales (style annotation/attribute)
- `EditAnywhere`, `BlueprintReadWrite`, `Category` en style constante
- `MYGAME_API` en style attribute
- Le reste du C++ colore normalement

## Comment Zed choisit entre C++ et Unreal C++ ?

| Priorite | Mecanisme | Portee |
|----------|-----------|--------|
| 1 (plus haute) | `file_types` dans `settings.json` | Projet ou global |
| 2 | `first_line_pattern` du `config.toml` | Automatique |
| 3 | Selection manuelle via la status bar | Fichier par fichier |

Le `first_line_pattern` de cette extension detecte automatiquement les fichiers commencant par :
- `// Copyright ... Epic`
- `// Fill out your copyright notice`
- `// Generated by UnrealBuildTool`

Pour tout autre cas, utilisez `file_types` dans les settings.

## Developper / Contribuer

### Structure du projet

```
unreal-cpp-zed/
├── extension.toml                      # Manifeste de l'extension Zed
├── LICENSE                             # MIT
├── README.md
├── .gitignore
└── languages/
    └── unreal_cpp/
        ├── config.toml                 # Declaration du langage
        ├── highlights.scm              # Coloration syntaxique (~370 lignes)
        ├── brackets.scm                # Paires de brackets
        ├── outline.scm                 # Symboles (classes, fonctions, namespaces)
        └── indents.scm                 # Regles d'indentation
```

### Modifier les queries

1. Editer les fichiers `.scm` dans `languages/unreal_cpp/`
2. Sauvegarder
3. Palette de commandes → **`zed: reload extensions`**
4. Les changements sont visibles immediatement

### Debug

- **Logs Zed** : palette → `zed: open log` pour voir les erreurs de parsing Tree-sitter
- **Mode verbose** : lancer Zed depuis le terminal avec `zed --foreground` pour le logging detaille (stdout/stderr)

### Comprendre highlights.scm

Le fichier est organise en 3 sections par ordre de priorite croissante (les patterns en fin de fichier ont priorite) :

1. **C base** — keywords, operateurs, types, fonctions (depuis tree-sitter-c)
2. **C++ additions** — templates, namespaces, keywords C++20 (depuis tree-sitter-cpp)
3. **Unreal Engine** — macros de reflexion, specifiers, assertions, logging

Pour ajouter un nouveau pattern UE, l'ajouter a la **fin** du fichier pour qu'il ait la priorite maximale.

### Ajouter une macro UE manquante

Exemple pour ajouter `MY_CUSTOM_MACRO(...)` :

```scheme
; a la fin de highlights.scm
(call_expression
  function: (identifier) @attribute
  (#match? @attribute "^MY_CUSTOM_MACRO$"))
```

### Captures disponibles

| Capture | Rendu thematique | Usage |
|---------|-----------------|-------|
| `@attribute` | Style annotation/decorateur | Macros UE de reflexion |
| `@constant` | Style constante | Specifier flags UE |
| `@function` | Style fonction | Appels + macros UE_LOG/check |
| `@function.special` | Style fonction speciale | TEXT(), preproc_function_def |
| `@keyword` | Style mot-cle | Keywords C/C++ |
| `@type` | Style type | Types, primitive_type |
| `@variable` | Style variable | Identifiants generiques |
| `@variable.special` | Style variable speciale | `this` |
| `@string` | Style chaine | Litteraux de chaine |
| `@string.escape` | Style echappement | `\n`, `\t`, etc. |
| `@comment` | Style commentaire | `//` et `/* */` |
| `@operator` | Style operateur | `+`, `->`, `==`, etc. |
| `@property` | Style propriete | Champs/membres |
| `@number` | Style nombre | Litteraux numeriques |
| `@boolean` | Style booleen | `true`, `false` |
| `@label` | Style label | Labels, modules |
| `@punctuation.bracket` | Style parenthese | `()`, `[]`, `{}` |
| `@punctuation.delimiter` | Style delimiteur | `.`, `;`, `,` |

## Evolutions prevues

### Snippets

Un dossier `snippets/` pourra etre ajoute avec des templates pour `uclass`, `ustruct`, `uprop`, `ufunc`, `ulog`, etc. Format : fichier JSON standard Zed dans `snippets/unreal_cpp.json`.

### Integration LSP / clangd

L'extension pourrait embarquer un adaptateur LSP Rust (via `zed_extension_api`) pour configurer automatiquement `clangd` avec les flags Unreal. Cela necessite un `Cargo.toml` + `src/lib.rs` implementant le trait `Extension`.

Config `.clangd` recommandee pour les projets Unreal en attendant :

```yaml
CompileFlags:
  Add:
    - "-ferror-limit=0"
    - "-Wno-unknown-pragmas"
    - "-Wno-unused-parameter"
  Remove:
    - "-W*"
Index:
  Background: Build
Diagnostics:
  UnusedIncludes: None
```

3. Ajouter l'entree dans `extensions.toml`
4. Ouvrir une PR
